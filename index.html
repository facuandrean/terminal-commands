¿Cómo crear un shellscript?

  1. Creo un archivo de texto con algún editor: nano, gedit, etc., o incluso touch y después lo abro con un editor. Ejemplo: gedit nombre-archivo.sh (es importante marcarlo como .sh)
  2. Necesitamos indicar que el intérprete de este archivo sea Bash. Entonces hay que abrir ese archivo (si es que se hizo con touch) y en la primera línea escribir: #!/bin/bash
  3. Luego se escriben distintas sentencias, y se guarda el programa.
  4. Para ejecutarlo, debemos darle permisos de ejecución, y una forma para hacerlo podría ser: chmod +x ./nombre-archivo.sh

Variables

Formas de creación de variables

  • Asignación directa: nombre_variable=contenido
  • Sustitución de comandos: el contenido de una variable es la salida de un comando. Hay distintas formas de hacerlo, una de ellas es: fecha=`date`. Recordar que cuando una variable guarda la salida de un comando, no se actualiza hasta que nosotros lo volvamos a actualizar.
  • Comando read: lee datos de la entrada (por teclado) y asigna cada campo ingresado (separado por un espacio) a una variable diferente. Ejemplo: Si nosotros ponemos read A B C D E, el script te va a dejar ingresar por teclado distintos datos; si nosotros ingresamos La flor es bella, cada espacio determina el fin del contenido de una variable y se van a asignar respectivamente, es decir, La va a ser el contenido de A, flor va a ser el contenido de B, y así sucesivamente. Si hay más variables que datos, se ignoran; y si hay más datos que variables, se ignoran.

Variables especiales

  • $?: Almacena el estado de salida del último comando ejecutado.
  • $!: Contiene el identificador del último proceso que se ejecutó en background.
  • $-: Contiene las opciones establecidas mediante el comando set.
  • $$: Contiene el identificador del proceso actual.
  • $#: Contiene el número de argumentos posicionales pasados al script.
  • $0: Nombre del shellscript.
  • $1, $2, $3, ..., $9: Posición de cada argumento posicional pasados al script. Ya a partir del 10 se usa entre paréntesis para no confundir $(10), $(11), etc.
  • $@: lista de argumentos posicionales pasados al script.

Variables de entorno de usuario

Contienen información del entorno del sistema para el usuario que ha iniciado sesión en ese momento.

  • HOME: directorio de usuario que inició sesión.
  • SHELL: ruta absoluta al intérprete de comandos.
  • USER: contiene el usuario logeado en la sesión actual.
  • PWD: ruta absoluta al directorio de trabajo del usuario.
  • PATH: lista de directorios que el shell examina al buscar ejecutables.
  • TERM: especifica la terminal por defecto.
  • PS1: identificador primario del shell (por defecto $).

Uso de variables

Una vez creada la variable, para usarla le debemos anteponer a su nombre, el símbolo de $. Por ejemplo:

                        
    # Creo la variable
    variable1=4

    # Uso la variable
    echo "El valor de la variable1 es: " $variable1

Podemos aplicar el mismo concepto con las variables de entorno de usuario. Por ejemplo: 

                        
    echo "La ruta absoluta al directorio de trabajo del usuario es: " $PWD

Comando test

Permite comprobar el valor de una expresión/sentencia devolviendo un 0 si la expresión es verdadera o un 1 si la expresión es falsa.

Test para validar archivos: 

                        
    test -bandera nombre-archivo
  • -e: existe el archivo.
  • -b: es especial de bloque.
  • -c: es especial de caracter.
  • -d: es un directorio.
  • -s: su tamaño es mayor que cero.
  • -f: es un archivo regular.
  • -r: existe y el usuario tiene permiso de lectura.
  • -w: existe y el usuario tiene permiso de escritura.
  • -x: existe y el usuario tiene permiso de ejecución.

Test para validar números: 

                        
    test numero1 operador numero2
  • "test numero1 -eq numero2": ambos enteros son iguales
  • "test numero1 -ne numero2": ambos enteros son distintos
  • "test numero1 -gt numero2": numero1 es mayor a numero2
  • "test numero1 -ge numero2": numero1 es mayor o igual a numero2
  • "test numero1 -lt numero2": numero1 es menor a numero2
  • "test numero1 -le numero2": numero1 es menor o igual a numero2

Formas de uso de test.

Se puede usar el comando "test": 
                        
    if test $edad -gt 18; 
    then
        echo "Eres mayor de edad."
    fi

Se puede usar "[ ]": 
                        
    if [ $edad -gt 18 ]; 
    then
        echo "Eres mayor de edad."
    fi

Notar que test y [ ] se usan para evaluar expresiones condicionales básicas. Son equivalentes pero la forma de los corchetes es más legible.



Se puede usar "[[ ]]":
                        
    if [[ $nombre == "Juan" ]]; 
    then
        echo "Hola, Juan."
    fi

Notar que [[ ]] proporciona funcionalidades adicionales. Es más flexible y puede manejar expresiones más complejas. Se pueden realizar comparaciones de cadenas sin preocuparte por la expansión de comodines.

                        
    if [[ $edad -gt 18 && $ciudad = "Barcelona" ]]; 
    then
        echo "Mayor de edad y vive en Barcelona."
    fi

Estructuras de control

Estructura IF - FI

Estructura simple
                        
    if sentencia
    then
        # si la sentencia es verdadera se ejecuta lo que haya en then    
        comando 1
        comando 2
        ...
        comando N
    fi
Estructura doble
                        
    if sentencia
        then
            # si la sentencia es verdadera se ejecuta lo que haya en then       
            comando 1
            comando 2
            ...
            comando N
        else
            # si la sentencia es falsa ignora el then y se ejecuta el else      
            comando 1
            comando 2
            ...
            comando N
    fi

Estructura CASE - ESAC

                        
    read opcion
    case $opcion in
        opcion1)
            echo "Se ejecutan las funcionalidades de la opción 1."
            ;;
        opcion2)
            echo "Se ejecutan las funcionalidades de la opción 2."
            ;;
        *)
            echo "Se ejecuta lo que queremos que pase en caso de que se ingrese una opción no definida anteriormente."      
            ;;
    esac

Estructura ciclo while

                        
    while condicion     
    do
        comando 1
        comando 2
        ...
        comando N
    done

Estructura ciclo for

                        
    for variable in lista_argumentos        
    do
        comando 1
        comando 2
        ...
        comando N
    done

Operaciones aritméticas

Usando la palabra reservada "let"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    let r=2+4
    echo $r

    # Para restar:
    let r=8-5
    echo $r

    # Para multiplicar:
    let r=2*4
    echo $r

    # Para división entera:
    let r=9/4
    echo $r

    # Para obtener el resto de la división:     
    let r=9%4
    echo $r

Notar que la división es entera, no arroja resultados decimales.

Usando el comando "expr"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    r=`expr 2 + 4`
    echo $r
    
    # Para restar:
    r=`expr 8 - 5`
    echo $r
    
    # Para multiplicar:
    r=`expr 2 \* 4`
    echo $r
    
    # Para división entera:
    r=`expr 9 / 4`
    echo $r
    
    # Para obtener el resto de la división:     
    r=`expr 9 % 4`
    echo $r

Notar que hay un espacio entre los operadores y operandos, esto es porque cada uno de ellos es un parámetro para el comando expr.

Usando la construcción "$(( x op y ))"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    r=$((2+4))
    echo $r

    # Para restar:
    r=$((8-5))
    echo $r

    # Para multiplicar:
    r=$((8*5))
    echo $r

    # Para división entera:
    r=$((9/4))
    echo $r

    # Para obtener el resto de la división:     
    r=$((9%4))
    echo $r

Notar que no hace falta el espacio entre los operadores y los operandos.
