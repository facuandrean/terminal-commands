¿Cómo crear un shellscript?

  1. Creo un archivo de texto con algún editor: nano, gedit, etc., o incluso touch y después lo abro con un editor. Ejemplo: gedit nombre-archivo.sh (es importante marcarlo como .sh)
  2. Necesitamos indicar que el intérprete de este archivo sea Bash. Entonces hay que abrir ese archivo (si es que se hizo con touch) y en la primera línea escribir: #!/bin/bash
  3. Luego se escriben distintas sentencias, y se guarda el programa.
  4. Para ejecutarlo, debemos darle permisos de ejecución, y una forma para hacerlo podría ser: chmod +x ./nombre-archivo.sh

Variables

Formas de creación de variables

  • Asignación directa: nombre_variable=contenido
  • Sustitución de comandos: el contenido de una variable es la salida de un comando. Hay distintas formas de hacerlo, una de ellas es: fecha=`date`. Recordar que cuando una variable guarda la salida de un comando, no se actualiza hasta que nosotros lo volvamos a actualizar.
  • Comando read: lee datos de la entrada (por teclado) y asigna cada campo ingresado (separado por un espacio) a una variable diferente. Ejemplo: Si nosotros ponemos read A B C D E, el script te va a dejar ingresar por teclado distintos datos; si nosotros ingresamos La flor es bella, cada espacio determina el fin del contenido de una variable y se van a asignar respectivamente, es decir, La va a ser el contenido de A, flor va a ser el contenido de B, y así sucesivamente. Si hay más variables que datos, se ignoran; y si hay más datos que variables, se ignoran.

Variables especiales

  • $? : Almacena el estado de salida del último comando ejecutado.
  • $! : Contiene el identificador del último proceso que se ejecutó en background.
  • $- : Contiene las opciones establecidas mediante el comando set.
  • $$ : Contiene el identificador del proceso actual.
  • $# : Contiene el número de argumentos posicionales pasados al script.
  • $0 : Nombre del shellscript.
  • $1, $2, $3, ..., $9: Posición de cada argumento posicional pasados al script. Ya a partir del 10 se usa entre paréntesis para no confundir $(10), $(11), etc.
  • $@ : lista de argumentos posicionales pasados al script.

Variables de entorno de usuario

Contienen información del entorno del sistema para el usuario que ha iniciado sesión en ese momento.

  • HOME: directorio de usuario que inició sesión.
  • SHELL: ruta absoluta al intérprete de comandos.
  • USER: contiene el usuario logeado en la sesión actual.
  • PWD: ruta absoluta al directorio de trabajo del usuario.
  • PATH: lista de directorios que el shell examina al buscar ejecutables.
  • TERM: especifica la terminal por defecto.
  • PS1: identificador primario del shell (por defecto $).

Uso de variables

Una vez creada la variable, para usarla le debemos anteponer a su nombre, el símbolo de $. Por ejemplo: 

                        
    # Creo la variable
    variable1=4

    # Uso la variable
    echo "El valor de la variable1 es: " $variable1

Podemos aplicar el mismo concepto con las variables de entorno de usuario. Por ejemplo: 

                        
    echo "La ruta absoluta al directorio de trabajo del usuario es: " $PWD

Comando test

Permite comprobar el valor de una expresión/sentencia devolviendo un 0 si la expresión es verdadera o un 1 si la expresión es falsa.

Test para validar archivos: 

                        
    test -bandera nombre-archivo
  • -e : existe el archivo.
  • -b : es especial de bloque.
  • -c : es especial de caracter.
  • -d : es un directorio.
  • -s : su tamaño es mayor que cero.
  • -f : es un archivo regular.
  • -r : existe y el usuario tiene permiso de lectura.
  • -w : existe y el usuario tiene permiso de escritura.
  • -x : existe y el usuario tiene permiso de ejecución.

Test para validar números: 

                        
    test numero1 operador numero2
  • "test numero1 -eq numero2": ambos enteros son iguales
  • "test numero1 -ne numero2": ambos enteros son distintos
  • "test numero1 -gt numero2": numero1 es mayor a numero2
  • "test numero1 -ge numero2": numero1 es mayor o igual a numero2
  • "test numero1 -lt numero2": numero1 es menor a numero2
  • "test numero1 -le numero2": numero1 es menor o igual a numero2

Formas de uso de test.

Se puede usar el comando "test": 
                        
    if test $edad -gt 18; 
    then
        echo "Eres mayor de edad."
    fi
Se puede usar "[ ]": 
                        
    if [ $edad -gt 18 ]; 
    then
        echo "Eres mayor de edad."
    fi

Notar que test y [ ] se usan para evaluar expresiones condicionales básicas. Son equivalentes pero la forma de los corchetes es más legible.

Se puede usar "[[ ]]":
                        
    if [[ $nombre == "Juan" ]]; 
    then
        echo "Hola, Juan."
    fi

Notar que [[ ]] proporciona funcionalidades adicionales. Es más flexible y puede manejar expresiones más complejas. Se pueden realizar comparaciones de cadenas sin preocuparte por la expansión de comodines.

                        
    if [[ $edad -gt 18 && $ciudad = "Barcelona" ]]; 
    then
        echo "Mayor de edad y vive en Barcelona."
    fi

Estructuras de control

Estructura IF - FI

Estructura simple
                        
    if sentencia
    then
        # si la sentencia es verdadera se ejecuta lo que haya en then    
        comando 1
        comando 2
        ...
        comando N
    fi
Estructura doble
                        
    if sentencia
        then
            # si la sentencia es verdadera se ejecuta lo que haya en then       
            comando 1
            comando 2
            ...
            comando N
        else
            # si la sentencia es falsa ignora el then y se ejecuta el else      
            comando 1
            comando 2
            ...
            comando N
    fi

Estructura CASE - ESAC

                        
    read opcion
    case $opcion in
        opcion1)
            echo "Se ejecutan las funcionalidades de la opción 1."
            ;;
        opcion2)
            echo "Se ejecutan las funcionalidades de la opción 2."
            ;;
        *)
            echo "Se ejecuta lo que queremos que pase en caso de que se ingrese una opción no definida anteriormente."      
            ;;
    esac

Estructura ciclo while

                        
    while condicion     
    do
        comando 1
        comando 2
        ...
        comando N
    done

Estructura ciclo for

                        
    for variable in lista_argumentos        
    do
        comando 1
        comando 2
        ...
        comando N
    done

Operaciones aritméticas

Usando la palabra reservada "let"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    let r=2+4
    echo $r

    # Para restar:
    let r=8-5
    echo $r

    # Para multiplicar:
    let r=2*4
    echo $r

    # Para división entera:
    let r=9/4
    echo $r

    # Para obtener el resto de la división:     
    let r=9%4
    echo $r

    # Para incrementar:
    let r++
    echo $r
    
    # Para decrementar:
    let r--
    echo $r

Notar que la división es entera, no arroja resultados decimales.

Usando el comando "expr"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    r=`expr 2 + 4`
    echo $r
    
    # Para restar:
    r=`expr 8 - 5`
    echo $r
    
    # Para multiplicar:
    r=`expr 2 \* 4`
    echo $r
    
    # Para división entera:
    r=`expr 9 / 4`
    echo $r
    
    # Para obtener el resto de la división:     
    r=`expr 9 % 4`
    echo $r    
    
    # Para incrementar:
    r=`expr $r + 1`
    echo $r
    
    # Para decrementar:
    r=`expr $r - 1`
    echo $r

Notar que hay un espacio entre los operadores y operandos, esto es porque cada uno de ellos es un parámetro para el comando expr.

Usando la construcción "$(( x op y ))"

Suma, resta, división (cociente y resto) y multiplicación

                        
    # Para sumar:
    r=$((2+4))
    echo $r

    # Para restar:
    r=$((8-5))
    echo $r

    # Para multiplicar:
    r=$((8*5))
    echo $r

    # Para división entera:
    r=$((9/4))
    echo $r

    # Para obtener el resto de la división:     
    r=$((9%4))
    echo $r

    # Para incrementar:
    ((r++))
    echo $r
    
    # Para decrementar:
    ((r--))
    echo $r

Notar que no hace falta el espacio entre los operadores y los operandos.

Menú de opciones

                        
    #!/bin/bash
    while true
    do
        echo "Menú de opciones:"
        echo "A) Ejecutar funcionalidad A."
        echo "B) Ejecutar funcionalidad B."
        echo "C) Salir."

        echo -n "Ingrese una opción: "
        read opcion
        case $opcion in
            a|A)
                # Acá se escribe el contenido de la funcionalidad A.     
                echo "Ejecutando la funcionalidad A..."
                ;;
            b|B)
                # Acá se escribe el contenido de la funcionalidad B.
                echo "Ejecutando la funcionalidad B..."
                ;;
            c|C)
                break;;
        esac

    done

Procesos.

Información de los procesos al usar el comando "ps -e".

                        
    F       |    S    |     UID     |   PID    |  PPID    |      C      |    PRI    |     NI     |    ADDR     |     SZ     |   WCHAN   |    TTY    |   TIME   |  CMD
    --------------------------------------------------------------------------------------------------------------------------------------------------------------------
    Flags   | Estados | Usuario     | ID del   | ID del   | Uso del     | Prioridad | Cambio de  | Dirección   | Tamaño del | Cambio de |  Terminal |  Tiempo  | Comando      
                      | que lanzó   | proceso. | proceso  | procesador. |           | prioridad. | de memoria. | proceso.   | contexto. |
                      | el proceso. |            padre.

Estados de un proceso en linux.

                        
    ID  |   Estado   | Descripción
    ----------------------------------------------------------------------------------------------
    S   | Sleep      | Suspendido. Con espera interrumpible (esperando que se complete un evento).      
        |            | Puede ser interrumpido por alguna señal.
    ----------------------------------------------------------------------------------------------
    D   | Sleep      | Suspendido. Con espera no interrumpible. (Generalmente espeando E/S).
    ----------------------------------------------------------------------------------------------
    T   | Stopped    | Detenido. Ha recibido una señal de detención.
    ----------------------------------------------------------------------------------------------
    R   | Running    | En ejecución. Solo existe un proceso utilizando la CPU que puede estar en
        |            | 2 modos:
        |            |     - Modo user: ejecuta instrucciones del programa de usuario.
        |            |     - Modo kernel: ejecuta instrucciones del kernel (llamadas al S.O.).
    ----------------------------------------------------------------------------------------------
    Z   | Zombie     | El proceso ha terminado pero no está muerto porque su estructura 
        |            | task_struct permanece referenciada desde el vector task. Es un proceso que
        |            | aún tiene dependencias.
    ----------------------------------------------------------------------------------------------
    X   | Dead       | Muerto (nunca debe ser visto).

Prioridades de procesos.

- 2 conceptos importantes: la prioridad PRI, es manejada por el kernel, y el valor de NI (nice), es manejado por el usuario.

- Fórmula de cálculo: PRI = 20 + NI donde NI >= -20 y NI <= 19.

- Mientras más chico el NI, más prioridad va a tener ese proceso.

- Podemos ver estas columnas con el comando ps -l o ps -el.

- Reiteramos: Al valor de PRI lo asigna el kernel al proceso (80 es el estándar), y al valor de NI lo modifica el usuario.

Agrupación de comandos que se pueden ejecutar en forma:

Secuencial

Se hace con el símbolo: ;
                        
    orden 1; orden 2; orden 3; ...; orden N

Condicional

Conectivo OR: ||
                        
    Ej: orden 1 || orden 2 -> mkdir dire1 || echo "No se pudo crear la carpeta dire1"
Conectivo AND: &&
                        
    Ej: orden 1 && orden 2 -> si no se ejecuta la primer orden, tampoco la segunda.

Grupo de órdenes

Podemos agrupar órdenes con el símbolo ; y con los paréntesis ( )
                        
    (ls /; pwd; who) > listado

Tuberías o pipelines

Se hace con el símbolo: |
                        
    orden 1 | orden 2 | orden 3 -> la salida de cada orden, se usa o se puede usar en la siguiente.

Forma de usar el comando Crontab.

El comando crontab permite ejecutar tareas recurrentes.

Para usarlo, debemos ejecutar en la línea de comandos: crontab -e.

Si es la primera vez, podemos elegir un editor para hacer que se abra.

Una vez abierto, nos permite guardar en un archivo los distintos comandos que queremos que se ejecuten cada cierto tiempo.

Podemos leer su documentación, pero se resume a esto: 

                        
    parámetro1 parámetro2 parámetro3 parámetro4 parámetro5 comando/s-a-ejecutar
  • Primer parámetro: significa cada cuántos minutos se va a ejecutar. Los valores posibles son 0-59.
  • Segundo parámetro: significa cada cuántas horas se va a ejecutar. Los valores posibles son 0-23.
  • Tercer parámetro: significa el día del mes en que se va a ejecutar. Los valores posibles son 0-31.
  • Cuarto parámetro: significa el numero del mes del año en que se va a ejecutar. Los valores posibles son 1-12.
  • Quinto parámetro: significa el dia de la semana en que se va a ejecutar. Los valores posibles son 0-7 (siendo 0 y 7 el día domingo).

Los valores de cada parámetro se pueden reemplazar por un aterisco *, el cual significa que se va a ejecutar cada valor de ese rango.

Ejemplo: 
                        
    10 * * * * ps -> la tarea se va a ejecutar a los 10 minutos de cada hora de cualquier día del año.      
    10 3 15 * * ls -li -> la tarea se ejecuta a las 3:10 del día 15 de cada mes.
    10 3 * * 1 top -> la tarea se ejecuta a las 3:10 de los días lunes.
Una notación especial para aclarar, es que si agregamos la barra / durante un parámetro, indica que se ejecuta cada cierto tiempo.
                        
    */10 * * * * ls -l / -> la tarea se ejecuta cada 10 minutos de cada hora, de cada día, etc.
Otra notación especial para aclarar es que si agregamos una coma durante un parámetro, la tarea se ejecuta en esos tiempos. Es como si fuese un "y".
                        
    0 2,3 * * * ls -l / -> la tarea se ejecuta a las 2 am y a las 3 am (porque el horario es de 24hs).

Discos virtuales

Pasos para crear un disco virtual

Aclaración, hay distintas formas de hacerlo, esta es una de ellas...

Ejecutamos el comando "dd": 

                        
    dd  if=/dev/zero of=nombre-disco-virtual.img bs=1M count=300

Explicación

  • dd: Es un comando utilizado para copiar y convertir datos. En este caso, lo estamos utilizando para crear un disco virtual.
  • if=/dev/zero: El argumento if (input file) especifica el archivo de entrada. En este caso, estás utilizando /dev/zero, que es un dispositivo especial en Linux que produce una secuencia constante de caracteres nulos (todos los bits en cero). Esto se utiliza para llenar el disco virtual con ceros.
  • of=nombre-disco-virtual.img: El argumento of (output file) especifica el archivo de salida. En este ejemplo, estás creando un archivo llamado nombre-disco-virtual.img que servirá como tu disco virtual.
  • bs=1M: El argumento bs (block size) especifica el tamaño de bloque para la operación de entrada/salida. En este caso, estás utilizando un tamaño de bloque de 1 megabyte.
  • count=300: Este argumento especifica cuántos bloques de tamaño bs se deben copiar desde la entrada al archivo de salida. Sería el espacio del disco virtual. En este ejemplo, estás creando un disco virtual de 300 megabytes, ya que estás copiando 300 bloques de 1 megabyte cada uno.

Crear particiones en disco virtual.

Para crear particiones en el disco virtual creado, ejecutamos el comando fdisk.

                        
    fdisk nombre-disco-virtual.img

Una vez dentro del disco virtual, podemos usar la opción de m para ver las distintas opciones posibles en su menú.

Una de las opciones es n, que es la que nos va a dejar crear particiones.

Si ingresamos n podemos ver que nos va a dejar crear particiones primarias o extendidas, y si creamos una extendida, la próxima va a ser lógica.

Si queremos crear particiones primarias, seguimos el orden del ingreso de datos. Ingresamos p y a partir de ahí nos va a pedir el orden de la partición (1-4), luego el punto de partida de espacio que va a empezar a ocupar esa partición (se pone el numero inicial del rango que muestra la terminal) y luego el espacio que va a ocupar esa partición (+numeroM, ej.: +15M).

Si queremos crear particiones extendidas, seguimos el mismo orden del ingreso de datos. Ingresamos e y a partir de ahí nos va a pedir la misma información. Un dato no menor es que la primera partición extendida debe ocupar un poco más que la suma de las particiones lógicas. Por ejemplo, si se quiere crear dos particiones extendidas, la primera de +20M y la segunda de +40M, su suma sería de +60M. La primera partición extendida se debe crear con un poco más del total de la suma de ambos tamaños, es decir, en vez de +20M, sería +65M para estar tranquilos. Esto es porque si creamos la primera partición extendida de +20M como necesitemos, no nos va a dejar crear la segunda partición lógica de +40M ya que no va a haber un espacio disponible debido a que +20M < +40M.

Con la opción p en el menú principal de fdisk podemos ver las particiones creadas.

No olvidar guardar los cambios en el disco virtual con la opción w antes de salir del menu de fdisk.

Asociar el disco virtual a un dispositivo de bucle

Una vez que has creado el disco virtual, necesitas asociarlo con un dispositivo de bucle para acceder a sus particiones. Ejecuta el siguiente comando:

                        
    losetup -fP nombre-disco-virtual.img

Nota: requiere ser super usuario.

Explicación:

  • losetup: Configura y controla los bucles del kernel de Linux para asociar archivos con dispositivos de bucle.
  • -f: Busca el primer bucle libre disponible.
  • -P: Detecta y configura automáticamente las particiones dentro del bucle.
  • nombre-disco-virtual.img: El nombre del archivo que contiene el disco virtual creado anteriormente.

Después de ejecutar este comando, podrás acceder a las particiones del disco virtual a través del dispositivo de bucle asociado.

Configuración de particiones.

Para saber el nombre del dispositivo de bucle asociado a la partición del disco virtual podemos ejecutar el comando sudo fdisk -l.

Crear un sistema de archivos ext3 en una partición.

                        
    mkfs.ext3 /dev/loopXpX

Nota: /dev/loopXpX es la partición o el dispositivo que formatearás con el sistema de archivos ext3.

Crear un sistema de archivos ext4 en una partición.

                        
    mkfs.ext4 /dev/loopXpX

Nota: /dev/loopXpX es la partición o el dispositivo que formatearás con el sistema de archivos ext4.

Configurar una partición o archivo como área de intercambio (swap).

                        
    mkswap /dev/loopXpX

Después de ejecutar mkswap, generalmente también se ejecuta el comando swapon para activar el área de intercambio.

                        
    swapon /dev/loopXpX

Montar particiones

Para montar particiones se hace con el comando mount.

                        
    mount /dev/loopXpX /ruta/a/donde/se/va/a/montar

Donde /dev/loopXpX representa la partición que deseas montar, y /ruta/a/donde/se/va/a/Montar es la ubicación en la que se montará la partición.

Nota: para una partición swap, se explicó arriba que se hace con swapon.

Desmontar particiones

Para desmontar particiones, lo podemos hacer con umount.

                        
    umount /dev/loopXpX

Donde /dev/loopXpX es la partición que deseas desmontar.
